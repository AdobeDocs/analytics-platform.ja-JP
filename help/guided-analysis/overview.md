# ガイド付き分析：概要

Adobe Product Analyticsを使用すると、製品チームは、Customer Journey Analyticsのクロスチャネルデータに基づいて構築されたガイド付き分析ワークフローを通じて、製品エクスペリエンスに関するデータとインサイトをセルフサービスできます。 ガイド付き分析は、製品チームがデータニーズにすばやく応え、高品質なインサイトを迅速に得て、より多くのデータ主導型の製品決定を下すことができるようにするレポート形式です。 部門をまたぐチームは、リアルタイムに接続して、これらのレポートを使用し、理解できます。

Analysis Workspaceおよびモバイルのスコアカードと同様、ガイド付き分析レポートでは、 [データビュー](../data-views/data-views.md)：を通じてAdobe Experience Platform内のデータを参照します。 [接続](../connections/overview.md). ガイド付き分析で作成したすべてのレポートは、Analysis Workspaceにシームレスに転送して、さらに調査をおこなうことができます。

ガイド付き分析では、データを分析および表示する複数の方法を提供します。 ビュータイプでは同じデータを異なる方法で表示できるので、同じイベントやセグメントを使用して様々なインサイトを得ることができます。 選択したビューに応じて、異なるクエリレールとビジュアライゼーション設定が表示されます。 ビューは自由に切り替えることができ、ビューがビューをサポートしている場合は、該当するクエリレールのコンポーネントが持ち越されます。

ガイド付き分析は、ビューのタイプを次のように分類します。 **分析タイプ**. 以下の分析タイプとビュータイプを使用できます。

| 分析タイプ | ビュータイプ | 説明 |
| --- | --- | --- |
| [!UICONTROL 影響] | [リリース](types/release.md) | リリース前とリリース後の同じ期間でパフォーマンスを比較します。 |
| [!UICONTROL 影響] | [初回使用](types/first-use.md) | 主要指標に対する初回の機能使用の影響を測定します。 |
| [!UICONTROL ファネル] | [摩擦](types/friction.md) | ステップ間のコンバージョン率を比較します。 |
| [!UICONTROL ファネル] | [コンバージョンの傾向](types/conversion-trends.md) | コンバージョン率の変化を経時的に追跡する。 |
| [!UICONTROL ユーザーの増加] | [アクティブ](types/active.md) | 誰が新規、定着、復帰、休止状態かを識別します。 |
| [!UICONTROL ユーザーの増加] | [純増加](types/net-growth.md) | ユーザーは増加していますか、それとも減少していますか？ |
| [!UICONTROL トレンド] | [用途](types/usage.md) | 時間の経過と共にユーザーエンゲージメントを測定します。 |
| [!UICONTROL トレンド] | [頻度](types/frequency.md) | 使用頻度でエンゲージメントを測定します。 |

## アクセス

組織でガイド付き分析がプロビジョニングされている場合は、Customer Journey Analyticsのホームページからアクセスできます。

1. クリック **[!UICONTROL ガイド付き分析]** ホームページから直接に移動します。 [使用状況トレンドビュー](types/usage.md).

   ![ランディングページタイル](assets/landing-page-tile.png)

1. クリック **[!UICONTROL 新規作成]** をクリックして別のビューオプションを表示し、解析の開始点を選択します。

   ![新しいモーダルの作成](assets/create-new-modal.png)

## インターフェイス

ガイド付き分析のインターフェイスは、質問と回答の形式に従っています。 クエリレールで質問を作成し、書き込まれたインサイト、グラフ、テーブルを使用して回答を取得します。 次の質問は、ビジュアライゼーションの設定と表示タイプを使用しておこなうことができます。

分析の種類に関係なく、ガイド付き分析では次の UI 要素が使用されます。

| インターフェイスのプレビュー | UI 要素 | 説明 |
| --- | --- | --- |
| ![クエリレール](assets/query-rail.png) | クエリレール | 分析を構成する目的のコンポーネント（イベント、プロパティおよびセグメント）を設定します。 各分析タイプでは、設定可能なイベント数とセグメント数に対して異なる制限を適用します。<p>フィルターアイコンを使用して、特定のイベントプロパティで絞り込むか、即座にセグメントを作成します。 プロパティを選択すると、「等しい」、「次を含む」、「次で終わる」などの標準フィルター条件に加えて、上位 1000 個のプロパティ値のリストをすばやくフィルタリングできます。<p>新しい解析タイプに切り替えた場合、クエリの選択は、その解析タイプで許可されている制限内に維持されます。 |
| ![グラフ](assets/chart.png) | グラフ | クエリレールと設定からの入力に基づいて返されるデータのビジュアライゼーション。 どのビジュアライゼーションが表示されるかは、グラフの上の表示と設定によって異なります。 また、グラフには次のものも含まれます。 <ul><li>**ツールチップ**：グラフのデータポイントの上にマウスポインターを置くと、ツールチップに詳細情報が表示されます。</li><li>**凡例**：グラフの凡例系列の上にマウスポインターを置くと、その系列に焦点を当て、他の系列を一時的に非表示にする、利用可能な定義が表示されます。 凡例内のシリーズをクリックすると、そのシリーズが非表示になります。</li><li>**注釈**：該当 [注釈](../components/annotations/overview.md) は、ビジュアライゼーションと凡例の間に表示されます。 これは、 ![注釈アイコン](assets/annotation.png) アイコンをクリックします。 時間の経過と共にデータを表示するビュータイプでは、 ![注釈アイコン](assets/annotation.png) アイコンをクリックします。 時間の経過と共にデータを表示しないタイプの表示では、 ![注釈アイコン](assets/annotation.png) アイコンをクリックします。</li><li>**クリックアクション**：任意のデータポイントを左クリックして、使用可能な次のアクションを表示します。 次のオプションがあります **セグメントを保存**.</li></ul> |
| ![テーブル](assets/table.png) | テーブル | クエリレールからの入力と設定に基づいて返されるデータのテーブル表現です。 テーブルの列は、グラフの上の表示タイプによって異なります。 この表には、次の項目も含まれます。 <ul><li>**クリックアクション**：系列を切り替えて、グラフの表示と非表示を切り替えます。 ![非表示アイコンを表示](assets/hide-in-chart.png) アイコンが各行に表示されます。 追加のアクションは、「 **[!UICONTROL その他]** メニュー。 次のオプションがあります **セグメントを保存**.</li></ul> |
| ![ビジュアライゼーション設定](assets/visualization-settings.png) | ビジュアライゼーション設定 | グラフの上にあるオプション。次の質問をし、グラフと表のデータの返し方をカスタマイズできます。 次のオプションは、すべてのビュータイプで使用でき、追加の設定はビューごとに使用できます。 <ul><li>**表示タイプ**：別の方法で特定の分析タイプのデータを表示できるドロップダウンセレクター。</li><li>**グラフ設定**：グラフやテーブルの表示を微調整します。 使用できるオプションは、選択した表示によって異なります。</li><li>**日付範囲**：分析の日付範囲を決定できるカレンダーピッカー。 また、日別、週別、月別など、トレンド表示の間隔を選択することもできます。</li><li>**インサイト**：表示する分析に応じたコンテキストインサイト。 矢印を使用してタブで追加のインサイトを表示したり、右上の電球アイコンを使用してこれらのインサイトを表示または非表示にしたりできます。</li></ul> |

## プロビジョニング

ガイド付き分析は、Adobe Product Analyticsの一部で、Customer Journey Analyticsへの有料アドオンです。 この一連の機能の使用を開始したい場合は、担当のAdobeアカウントチームにお問い合わせください。

ガイド付き分析を使用するよう組織をプロビジョニングすると、製品プロファイル管理者は、Adobe Admin Consoleでその組織へのアクセスを追加または削除できます。

1. にログインします。 [Adobe Admin Console](https://adminconsole.adobe.com).
1. 選択 **[!UICONTROL Customer Journey Analytics]** 」をクリックします。
1. 編集する権限に対して、目的の製品プロファイルを選択します。
1. 次をクリック： **[!UICONTROL 権限]** 「 」タブで、「 」をクリックします。 **[!UICONTROL 編集]** under [!UICONTROL レポートツール].
1. の横のプラスアイコンをクリックします。 **[!UICONTROL ガイド付き分析アクセス]** 次のリスト： [!UICONTROL 使用可能な権限項目]をクリックし、 [!UICONTROL 含まれる権限項目].
1. 「**[!UICONTROL 保存]**」をクリックします。

>[!TIP]
>
>管理者によっては、ガイド付き分析を有効にし、新規ユーザーのAnalysis Workspaceを無効にしてCustomer Journey Analyticsを希望する場合があります。 これらのユーザーが製品と組織のデータで成熟したら、Analysis Workspaceへのアクセスを有効にできます。
