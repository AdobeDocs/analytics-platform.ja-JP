# ガイド付き分析：概要

{{release-limited-testing}}

ガイド付き分析は、製品チームがデータニーズにすばやく応え、高品質なインサイトを迅速に得て、より多くのデータ主導型の製品決定を下すことができるようにするレポート形式です。 部門をまたぐチームは、リアルタイムに接続して、これらのレポートを使用し、理解できます。

Analysis Workspaceやモバイルのスコアカードと同様、ガイド付き分析レポートでは [データビュー](../data-views/data-views.md)：を通じてAdobe Experience Platform内のデータを参照します。 [接続](../connections/overview.md). ガイド付き分析で作成したすべてのレポートは、Analysis Workspaceにシームレスに転送して、さらに調査をおこなうことができます。

ガイド付き分析では、データを分析および表示する複数の方法を提供します。 ビュータイプでは同じデータを異なる方法で表示できるので、同じイベントやセグメントを使用して様々なインサイトを得ることができます。 選択したビューに応じて、異なるクエリレールとビジュアライゼーション設定が表示されます。 ビューは自由に切り替えることができ、ビューがビューをサポートしている場合は、該当するクエリレールのコンポーネントが持ち越されます。

ガイド付き分析は、ビューのタイプを次のように分類します。 **分析タイプ**. 以下の分析タイプとビュータイプを使用できます。

| 分析タイプ | ビュータイプ | 説明 |
| --- | --- | --- |
| [!UICONTROL 影響] | [リリース](types/release.md) | リリース前とリリース後の同じ期間のパフォーマンスを比較. |
| [!UICONTROL 影響] | [初回使用](types/first-use.md) | 機能の初回使用が主要指標に与える影響を測定. |
| [!UICONTROL ファネル] | [摩擦](types/friction.md) | ステップ間のコンバージョン率を比較します. |
| [!UICONTROL ファネル] | [換算傾向](types/conversion-trends.md) | 換算率の経時的な変化を追跡. |
| [!UICONTROL ユーザーの増加率] | [アクティブ](types/active.md) | 新規、継続、復帰または休眠ユーザーを特定. |
| [!UICONTROL 純増加率] | [純増加率](types/net-growth.md) | ユーザーは増加していますか、それとも減少していますか？ |
| [!UICONTROL 傾向] | [用途](types/usage.md) | ユーザーエンゲージメントの推移を測定します. |

{style="table-layout:auto"}

## アクセス

組織でガイド付き分析がプロビジョニングされている場合は、Customer Journey Analyticsのホームページからアクセスできます。

1. クリック **[!UICONTROL ガイド付き分析]** ホームページから直接 [使用状況トレンドビュー](types/usage.md).

   ![新しいモーダルの作成](assets/create-new-modal.png)

1. クリック **[!UICONTROL 新規作成]** をクリックして別のビューオプションを表示し、解析の開始点を選択します。

   ![ランディングページタイル](assets/landing-page-tile.png)

## インターフェイス

分析タイプに関係なく、ガイド付き分析のインターフェイスは、次の主な UI 要素で構成されます。

| インターフェイスのプレビュー | UI 要素 | 説明 |
| --- | --- | --- |
| ![クエリレール](assets/query-rail.png) | クエリレール | 分析を構成する目的のコンポーネント（イベント、プロパティおよびセグメント）を設定します。 各分析タイプでは、設定可能なイベント数とセグメント数に対して異なる制限を適用します。 新しい解析タイプに切り替えた場合、クエリの選択は、その解析タイプで許可されている制限内に維持されます。 |
| ![グラフ](assets/chart.png) | グラフ | クエリレールと設定からの入力に基づいて返されるデータのビジュアライゼーション。 どのビジュアライゼーションが表示されるかは、グラフの上の表示と設定によって異なります。 使用可能なビューは、クエリレールの上にある分析タイプによって異なります。 また、グラフには次のものも含まれます。 <ul><li>**ツールチップ**:グラフのデータポイントの上にマウスポインターを置くと、ツールチップに詳細情報が表示されます。</li><li>**凡例**:グラフの凡例の上にマウスポインターを置くと、系列の定義が表示されます（使用可能な場合）。</li><li>**クリックアクション**:任意のデータポイントを左クリックして、使用可能な次のアクションを表示します。 次のオプションがあります **セグメントを保存**.</li></ul> |
| ![テーブル](assets/table.png) | テーブル | クエリレールからの入力と設定に基づいて返されるデータのテーブル表現です。 テーブルの列は、グラフの上の表示タイプによって異なります。 使用可能なビューは、クエリレールの上にある分析タイプによって異なります。 この表には、次の項目も含まれます。 <ul><li>**クリックアクション**:次に使用可能なアクションを表示するには、 **[!UICONTROL 詳細]** メニュー 次のオプションがあります **セグメントを保存**.</li></ul> |
| ![ビジュアライゼーション設定](assets/visualization-settings.png) | ビジュアライゼーション設定 | グラフの上にあるいくつかのオプション。グラフと表のデータの返し方をカスタマイズできます。<ul><li>**表示タイプ**:別の方法で特定の分析タイプのデータを表示するためのドロップダウンセレクター。</li><li>**グラフ設定**:グラフとテーブルの表示を微調整する。 使用できるオプションは、選択した表示によって異なります。</li><li>**日付範囲**:分析の日付範囲を決定するためのカレンダーピッカー。 また、日別、週別、月別など、トレンド表示の間隔を選択することもできます。</li><li>**インサイト**:表示する分析に応じたコンテキストインサイト。 矢印を使用してタブで追加のインサイトを表示したり、右上の電球アイコンを使用してこれらのインサイトを表示または非表示にしたりできます。</li></ul> |
| ![メニュー](assets/menu.png) | メニュー | ガイド付き分析の右上にあるコマンドで、分析に関する包括的なアクションを提供します。<ul><li>**データビューセレクター**:分析で使用されるデータビューを変更します。 データビューを変更すると、クエリレールの使用可能なコンポーネントも変更されます。</li><li>**保存**:解析を保存します。 新しい分析を保存する場合、名前と説明をリクエストするモーダルウィンドウが表示されます。</li><li>**名前を付けて保存**:解析を現在の解析とは別に保存し、コピーを作成します。 新しい名前と説明をリクエストするモーダルウィンドウが表示されます。</li><li>**ワークスペースで開く**:Analysis Workspaceで現在のガイド付き分析を再作成します。 Workspace プロジェクトは新しいタブで作成され、ガイド付き分析内で作業中に中断するのを防ぎます。 解析のコピーであり、開いた後は元のガイド付き分析と同期しません。 このコマンドは、アナリストチームに引き渡す場合や、ガイド付き分析で可能なデータよりも詳細にデータを分析する場合に使用します。</li><li>**クリップボードにコピー**:グラフのグラフィックをクリップボードにコピーして、他のアプリケーションに貼り付けます。 クエリレールとテーブルは、グラフィックに含まれていません。</li><li>**PNG をダウンロード**:グラフのグラフィックを `.png`. クエリレールとテーブルは、グラフィックに含まれていません。</li><li>**CSV をダウンロード**:テーブルデータを `.csv`. クエリレールとグラフはファイルに含まれていません。</li></ul> |

## プロビジョニング

ガイド付き分析は、Adobe Product Analyticsの一部で、Customer Journey Analyticsへの有料アドオンです。 この一連の機能の使用を開始したい場合は、担当のAdobeアカウントチームにお問い合わせください。

ガイド付き分析を使用するよう組織をプロビジョニングすると、製品プロファイル管理者は、Adobe Admin Consoleでその組織へのアクセスを追加または削除できます。

1. にログインします。 [Adobe Admin Console](https://adminconsole.adobe.com).
1. 選択 **[!UICONTROL Customer Journey Analytics]** を選択します。
1. 編集する権限に対して、目的の製品プロファイルを選択します。
1. 次をクリック： **[!UICONTROL 権限]** 「 」タブで、「 **[!UICONTROL 編集]** under [!UICONTROL レポートツール].
1. の横のプラスアイコンをクリックします。 **[!UICONTROL ガイド付き分析アクセス]** リスト内 [!UICONTROL 使用可能な権限項目]をクリックし、 [!UICONTROL 含まれる権限項目].
1. 「**[!UICONTROL 保存]**」をクリックします。

>[!TIP]
>
>管理者によっては、ガイド付き分析を有効にし、新規ユーザーのAnalysis Workspaceを無効にしてCustomer Journey Analyticsを希望する場合があります。 これらのユーザーが製品と組織のデータで成熟したら、Analysis Workspaceへのアクセスを有効にできます。
