## ドリルダウンする

Power BIでは、[&#x200B; ドリルモード &#x200B;](https://learn.microsoft.com/en-us/power-bi/consumer/end-user-drill)をサポートしており、特定のビジュアライゼーションに関する詳細を確認できます。 次の例では、製品カテゴリの購買収益を分析します。 製品カテゴリを表すバーのコンテキストメニューから、**[!UICONTROL ドリルダウン]**&#x200B;を選択できます。

![Power BI ドリルダウン &#x200B;](../assets/uc15-powerbi-drilldown.png)

ドリルダウンすると、選択した製品カテゴリ内の製品の購入収益でビジュアライゼーションが更新されます。

![Power BI ドリルアップ &#x200B;](../assets/uc15-powerbi-drillup.png)

ドリルダウンすると、次のSQL クエリが表示され、`WHERE`句が使用されます。

```sql
select "_"."product_category" as "c25",
    "_"."product_name" as "c26",
    "_"."a0" as "a0"
from 
(
    select "_"."product_category",
        "_"."product_name",
        "_"."a0"
    from 
    (
        select "_"."product_category",
            "_"."product_name",
            "_"."a0"
        from 
        (
            select "rows"."product_category" as "product_category",
                "rows"."product_name" as "product_name",
                sum("rows"."purchase_revenue") as "a0"
            from 
            (
                select "_"."product_category",
                    "_"."product_name",
                    "_"."purchase_revenue"
                from "public"."cc_data_view" "_"
                where ("_"."daterange" >= date '2023-01-01' and "_"."product_category" = 'Fishing') and "_"."daterange" < date '2024-01-01'
            ) "rows"
            group by "product_category",
                "product_name"
        ) "_"
        where not "_"."a0" is null
    ) "_"
) "_"
order by "_"."product_category",
        "_"."product_name"
limit 1001
```

## ドリルダウンする

Tableauは、[&#x200B; ドリルモード &#x200B;](https://learn.microsoft.com/en-us/power-bi/consumer/end-user-drill) ～ [階層](https://help.tableau.com/current/pro/desktop/en-us/qs_hierarchies.htm)をサポートしています。 次の例では、**[!UICONTROL テーブル]**&#x200B;内の&#x200B;**[!UICONTROL 製品名]** フィールドを選択し、**[!UICONTROL 製品カテゴリ]**&#x200B;の上にドラッグすると、階層が作成されます。 次に、製品カテゴリを表すバーのコンテキストメニューから、「**[!UICONTROL + ドリルダウン]**」を選択できます。

![Tableau ドリルダウン &#x200B;](../assets/uc15-tableau-drilldown.png)

ドリルダウンすると、選択した製品カテゴリ内の製品の購入収益でビジュアライゼーションが更新されます。

![Tableau ドリルアップ &#x200B;](../assets/uc15-tableau-drillup.png)

ドリルダウンすると、GROUP BY句を使用している次のSQL クエリが表示されます。

```sql
SELECT CAST("cc_data_view"."product_category" AS TEXT) AS "product_category",
  CAST("cc_data_view"."product_name" AS TEXT) AS "product_name",
  SUM("cc_data_view"."purchase_revenue") AS "sum:purchase_revenue:ok"
FROM "public"."cc_data_view" "cc_data_view"
WHERE (("cc_data_view"."daterange" >= (TIMESTAMP '2023-01-01 00:00:00.000')) AND ("cc_data_view"."daterange" < (TIMESTAMP '2024-01-01 00:00:00.000')))
GROUP BY 1,
  2
```

クエリでは、**not**&#x200B;は選択した製品カテゴリに結果を制限しません。選択した製品カテゴリを表示するのはビジュアライゼーションのみです。

![Tableau ドリルアップ &#x200B;](../assets/uc15-tableau-drillup2.png)

または、1つのビジュアルが別のビジュアルの選択の結果であるドリルダウンダッシュボードを作成することもできます。 次の例では、**[!UICONTROL 製品カテゴリ]**&#x200B;のビジュアライゼーションをフィルターとして使用して、**[!UICONTROL 製品名]** テーブルを更新しています。 このビジュアライゼーションフィルターはクライアント専用で、追加のSQL クエリは発生しません。

![Tableau ビジュアライゼーションフィルター](../assets/uc15-tableau-visualizationfilter.png)
