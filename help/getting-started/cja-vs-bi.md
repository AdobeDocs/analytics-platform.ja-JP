# CJA と BI ソリューションの比較

現在、カスタマーエクスペリエンスに重点を置いているので、ブランドは、全体的なカスタマージャーニーをより深く理解するために、高度なソリューションを必要としています。 この完全なカスタマージャーニーを理解すると、オンラインおよびオフラインのチャネルが顧客にどのように関与し、コンバージョン、定着、ロイヤルティを高めるかに関する貴重なインサイトを分析し、得ることができます。 お客様の遍歴は、すしフードチェーンでの食事のオンラインでの簡単な順序です。 または、顧客がオンライン調査とディーラーショールームへの訪問、および最終的な対面購入を組み合わせた新しい車の購入。

多くの組織では、オムニチャネルデータをデータレイクまたはデータウェアハウスに統合しています。 ビジネスインテリジェンス (BI) ツールは、これらのデータストアの上に使用して、カスタマージャーニーを理解するためにビジネスで必要なレポート、ビジュアライゼーション、インサイトを提供します。 多くの場合、このソリューションとツールの組み合わせは、自然と設計によって汎用的であり、お客様に明示的に焦点を当てていません。 Customer Journey Analytics(CJA) は、マーケター、データアナリスト、データサイエンティストなど、顧客体験に責任を持つ人々を強化することに重点を置いています。 このツールを使用すると、他の多くの BI ツールに制限なく、すべてのチャネルにわたるフルコンテキストでカスタマージャーニーをリアルタイムに視覚化できます。

ドキュメントのこの節では、まず上記の目的を達成するために使用される一般的なワークフローを見て、CJA と一般的に使用される BI ツールの基本的な違いを説明します。カスタマージャーニーについて理解します。 次に、CJA ツールと BI ツールでデータの保存、収集およびクエリの方法に関する詳細を提供します。 最後に、ビジュアライゼーション機能の違いについて説明します。

## 従来の BI ワークフロー

従来のカスタマージャーニー分析のアプローチによくある障害は、それが顧客中心ではないことです。 各チームは、サイロでデータを収集し、アクセス権のあるデータに基づいて、エクスペリエンスを分析および最適化します。

![一般的な BI ワークフロー](./assets/biworkflow.png)

特定のデジタルキャンペーンが異なるデータサイロに格納されたオフラインアクションに与える影響を理解するには、BI チームのキューにリクエストを発行します。 BI チームは、データを取得して変換するために必要なクエリを書き込みます。 生データが取得されると、BI チームはビジュアライゼーションを作成します。 データは共有され、他のシステムでアクティベーションするためにインサイトを参照し、データを抽出する時間を費やします。

これらの各手順には、数時間、数日、または数週間かかる場合があります。 フォローアップの質問や問題がクエリされたデータに含まれている場合は、それらの質問に対処する前にさらに時間がかかり、サイクルが継続する可能性があります。 継続的な分析、カスタマージャーニーの調査、理解に関しては、このプロセスは非効率で拡張性に欠けます。 また、BI チームは、通常、カスタマージャーニーに関する質問以外にも対処します。

## CJA:オンラインおよびオフラインのデータに関するワークフローを民主化

CJA は、カスタマージャーニーを理解することのみを目的として、オンラインとオフラインのクロスチャネルデータを包括的な顧客レベルで接続する環境を提供します。 次の操作を行うために初期設定が必要です。 [接続](/help/connections/overview.md) および [ビューを定義](/help/data-views/data-views.md) を関連性の高いと見なすデータに追加します。 ただし、一旦完了すると、そのデータは継続的な分析や調査にすぐに使用できます。 カスタマージャーニーに関するインサイトを徐々に得て理解できます。 オンラインとオフラインのデータを組み合わせて民主化することで、カスタマージャーニーに関する質問に数秒で答えることができます。

![CJA ワークフロー](./assets/cjaworkflow.png)

CJA を使用して、Visual Analysis Workspace環境を使用して質問し、ほぼ瞬時にインサイトを得ることができます。 クロスチャネルのデータとレポートは直ちに使用可能になり、SQL コードは不要です。 追加のクエリや分析は、完全に相関関係のあるデータを使用して、UI にシンプルなドラッグ&amp;ドロップで実行できます。 引き続き質問をし、必要に応じて詳細を段階的に調べることができます。 その後、アクティベーションやオーケストレーション用のオーディエンスを共有するなど、発見したインサイトに対して即座に対処できます。

## CJA の強力なレポートエンジン

CJA は、数百台または数千台のサーバーにわたって分析を分散し、数秒以内にAnalysis Workspaceにデータを表示する、強力な独自のアーキテクチャを使用しています。 この処理アーキテクチャの主なプロパティには、次のものがあります。

* **個々の顧客関連クエリに最適化**:技術的には、CJA はキャッシュを広範に利用する分散型レポートエンジンにデータを保存します。 このエンジンは、個々のレベルのイベントデータに対するレスポンシブクエリに合わせて微調整され、顧客関連のクエリに対して完全に最適化されています。 レポートエンジンは、集計指標を即座に計算できるように、列指向のビットマップインデックスにデータを格納します。 強力なセグメント化/オーディエンス分析を可能にする広範なフィルタリングエンジンを備えています。 また、データポイント間のシーケンスに関するコアな理解を持っており、これらのデータポイント間の動作（発生した順序）の分析や、様々な複雑なモデルを使用したアトリビューションの割り当てに役立ちます。

* **複雑なパスおよびフィルターの迅速な適用**:レポートエンジンは、部分的に並べ替えられた階層データセット（ユーザー/セッション/イベントなど）で機能します。 最上位レベルのオブジェクト（個々のプロファイル）のすべてのデータは、正確な結果を得るために、1 つの処理ノード上に存在します。 このパーティション化により、複雑なパスやフィルターを迅速に適用できます。 セッション化、アトリビューション、データ属性のステートフル永続性、複雑なデータ操作オプションなどの複雑な操作は、高速なレポート時間で大規模に実行されます。 BI 世界では、通常、このような操作には、各使用例に対して新しい OLAP キューブを作成する必要があります。 CJA のレポートエンジンを使用すると、すべてのクエリでデータセット全体に自由にアクセスでき、完全に相関したデータになり、事前に手を加える必要がなくなります。

* **複雑なデータストリームの効率的なクエリ**:従来の SQL および NoSQL データベースとの最大の違いの 1 つは、シーケンス指向の関係に基づいて、基本レベルで述語を決定する機能です。 これらの基本的なクエリ操作は、多数のインターリーブされた（さらにはネストされた）シーケンスで構成されたレコードストリームを調べることができます。 連続した単一のシーケンス操作の効率で、これらの相互に関係するデータストリームすべてに対してクエリを実行します。

* **大規模なクエリに迅速に対応するように設計されています**:レポートエンジンは、従来のビッグデータシステムほど一般的な目的ではありません。 ただし、この特別な設計は、通常、1 秒未満で、数百万件または数十億件のレコード（イベントデータ/エクスペリエンスイベント）にわたるクエリに回答するように設計されています。 他のビッグデータシステムとは異なり、データをサンプリングしたり、考えるすべての質問に対する回答を事前に計算したりして、これを行うわけではありません。 代わりに、インタラクティブなクエリの使用例をサポートするのに十分な速さで回答を計算できます。 CJA のレポートエンジンがこの特定の設計により、継続的な分析や調査に対して、データを容易かつ高速に利用できるので、カスタマージャーニーに関するインサイトと理解を徐々に深めることができます。

* **ヘッドレス BI ソリューションとしての役割を果たす**:ディメンション、指標、フィルターを 1 か所で定義すると、CJA クライアント（アドビのパブリック CJA API を含む）はすべて、これらのコンポーネントにアクセスできます。 これにより、複雑なクエリがエンドユーザーから抽象化され、どのレポートやビジュアライゼーションクライアントを使用しても、結果が同じであることが保証されます。

## CJA 独自のビジュアライゼーション機能

レポートエンジンは、CJA がそのレポートエンジン内のすべてのカスタマージャーニーデータに対し、徐々にやり取りし、対応できるようにするのに不可欠です。 CJA には、視覚的およびドラッグ&amp;ドロップによってこの作業を実行できる、広範なコンポーネントセットが付属しています。 BI ビジュアライゼーションツールを使用すると、SQL で準備されたデータの境界内を（IT で定義されたとおりに）調査できます。 CJA を使用すると、IT に戻って別の SQL ビューを構築する必要なく、必要な分類やスライス&amp;ダイスをおこなうことができます。

「プログレッシブ」は、ここでの重要な概念です。BI ツールのほとんどのビジュアライゼーションとは異なり、CJA の視覚的なドラッグ&amp;ドロップ UI を使用すると、次のように、特定のニーズに合わせてデータを継続的に分類できます。関連する指標、ディメンション、フィルター（セグメント）、計算、タイムライン、注釈、その他の分析値を使用して、視覚的なクエリをインタラクティブに作成できます。

これらのビジュアライゼーションコンポーネントに組み込まれているスマート機能は次のとおりです。

* **仮想アナリストの機能** 例： [異常値検出](/help/analysis-workspace/virtual-analyst/c-anomaly-detection/anomaly-detection.md) 予測アルゴリズムと機械学習を使用して、データにおいて異常な行動を引き起こしているものに関するインサイトを提供する

* **高度な分析機能** 顧客のジャーニーに関するインサイトに特に焦点を当てたもの（例： ） [フロー図](/help/analysis-workspace/visualizations/c-flow/flow.md), [attribution IQ](/help/analysis-workspace/attribution/overview.md), [フォールアウト図](/help/analysis-workspace/visualizations/fallout/fallout-flow.md)、および [ディメンション分類](/help/components/dimensions/t-breakdown-fa.md). 標準のビジュアライゼーションの例を次に示します。

   * [コホート/待ち時間テーブルによる顧客維持分析](/help/analysis-workspace/visualizations/cohort-table/cohort-use-cases.md)では、指標やディメンションをビルダーにドラッグ&amp;ドロップするだけで、30 秒未満で実行できます。

   * [フォールアウト](/help/analysis-workspace/visualizations/fallout/configuring-fallout.md) / [流れ](/help/analysis-workspace/visualizations/c-flow/create-flow.md) ビジュアライゼーション。 1 分以内に設定します。

   * [アトリビューションモデル](/help/analysis-workspace/attribution/algorithmic.md) ファーストタッチ、ラストタッチ、パーティシペーション、タイムディケイと同様に、設定に数回クリックするカスタムのものも含まれます。

* **プログレッシブ調査の各ステップでのセグメント化機能**:ユーザーが意味を持つと思われる場合は、オーディエンスをExperience Platformに公開し、そこから、サポートされる任意の宛先に公開できます。

* **セッション化** 完全に [カスタマイズ可能](/help/data-views/component-settings/persistence.md):いつセッションを、カスタマージャーニーのチャネルの一部として、開始および終了するかを決定します。

* **キュレーションと民主化**:CJA で作成されるダッシュボードは次のとおりです。

   * [キュレート済み](/help/analysis-workspace/curate-share/curate.md) 組織内の他の個人に対し、継続的な調査を行う
   * を使用して Excel にエクスポート [Report Builder](/help/report-builder/report-buider-overview.md) （専用のプラグイン）
   * [共有](/help/analysis-workspace/curate-share/share-projects.md) 様々な形式の [PDF](/help/analysis-workspace/curate-share/download-send.md), [CSV](/help/analysis-workspace/curate-share/download-send.md) そして、 [専用モバイルアプリ](/help/mobile-app/home.md)を、最終レポートやビジュアライゼーションに関心のあるユーザーに追加できます。

CJA のビジュアライゼーション機能と BI ツールのオファーを比較するのは、使用可能な様々なビジュアライゼーションなので困難です。 一部の BI ツールにはより高度なビジュアライゼーションが用意されていますが、CJA は、インタラクティブで相互運用可能なカスタマージャーニーのビジュアライゼーションに注力しており、追加クエリごとに課金をおこなうことなく、数秒でデータを分類できます。


## 概要

CJA は、BI ツールとは異なり、高度に最適化されたカスタマージャーニーに焦点を当てたレポートエンジンを、使いやすいツールやコンポーネントとシームレスに統合し、分析を実行し、レポートを作成し、高度なビジュアライゼーションを行います。 クエリエンジンとビジュアライゼーション環境の間を行き来する必要なく、1 つの UI 内からすべてを実行できます。
