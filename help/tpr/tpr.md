# 合計母集団レポート

総母集団レポートでは、プロファイルおよびルックアップデータセットで定義されたエンティティと、イベントデータセットからの時間ベースの一連のイベントを分析およびレポートする機能が導入されています。 この能力により、ビジネスの顧客基盤のあらゆる範囲を反映する、新しいクラスのクエリ、指標、オーディエンス定義が可能になります。

Customer Journey Analyticsは、イベントを中心に構築されています。 すべての指標、すべてのビジュアライゼーション、すべてのパネル、すべてのレポートは、日付時間範囲と、その日付時間範囲内に発生するイベントにアンカーされます。 ソリューションに関する次のような質問があります。

| 質問 | イベント | 日付時間範囲 |
|---|---|---|
| 先週購入した人はいますか？ | 購入 | 先週 |
| 第1四半期に価格ページを訪問したアカウント数は？ | 訪問 | Q1 |

ワークスペースパネルの日付範囲はデータをフィルタリングし、ディメンションと指標はその日付範囲で何が起こったのかを示します。

しかし、企業が取り組む必要があるあらゆる質問が、実際に起こったことに関するものであるとは限りません。 時には、自分の家族について知る必要もあります。

| 質問 | イベント | 日付時間範囲 |
|---|---|---|
| アクティブな顧客は何人いますか？ | 該当なし | なし |
| データベースのアカウント数は？ | 該当なし | なし |
| 過去30日間に購入していない会員の割合？ | 該当なし | 過去30日間 |

これらの質問はイベントに関するものではなく、これらの人々やアカウントが最近何かをしてきたかどうかにかかわらず、存在する人々やアカウントに関するものです。

総母集団レポートでは、プロファイルデータをレポートする新しい指標クラスが導入されます。 プロファイルデータセット内の人物とアカウントを含めることができるプロファイルデータは、パネルの日付範囲とは独立しています。 合計母集団レポートを使用すると、母集団ベースの指標とイベントベースの指標を同じ分析に組み合わせることができ、顧客が誰であるか、顧客が何をしたかをより包括的に把握できます。

合計母集団レポートを使用すると、Customer Journey Analyticsは、イベントアクティビティに関係なく、プロファイルおよびルックアップデータセットで定義されたすべてのエンティティをレポートできます。 レポートには次のものが含まれます。

* **プロファイルベースのクエリ**：プロファイル（すべての人物、アカウント、商談、購買グループ）の属性（イベントに関係なく）を分析します。
* **プロファイルからイベントクエリを差し引く**：レポートウィンドウ中に特定のアクションやエクスペリエンスを実行しなかったプロファイル（すべての人物、アカウント、商談、購買グループ）を特定します。
* **共有ルックアップ**：取り込みコストを削減し、パフォーマンスを向上させるために、複数のエンティティをまたいでルックアップデータセットの再利用をサポートします。

## 母集団レポート指標の合計

合計母集団レポート指標は、Customer Journey Analyticsで一般的に使用される指標とは異なる動作します。

* 合計母集団レポート指標は、パネルの日付範囲にバインドされません。 **[!UICONTROL 合計人数（プロファイル）]**&#x200B;のような合計母集団レポート指標は、パネルに適用された日付範囲に関係なく、プロファイルデータセットから現在の母集団を返します。 日付フィルターと日付範囲の比較は、これらのフィルターと比較がイベント指標に与える影響は、母集団レポート指標の合計には影響しません。
* 合計母集団レポートには、接続にプロファイルデータセットが必要です。 合計母集団レポート指標は、接続に1つ以上のプロファイルデータセットと1つ以上のイベントデータセットが含まれている場合にのみ表示されます。 イベントデータセットのみの接続は、以前と同じように引き続き機能し、合計母集団レポート指標は表示されません。

Workspaceでは、母集団指標の合計に明確なアイコン（TBD）がマークされるので、パネルの日付範囲を尊重する指標とそうでない指標をすばやく特定できます。 総母集団指標は、ほとんどの場所でイベント指標と一緒に使用できますが、イベントシーケンス（フォールアウトやフローなど）に依存するビジュアライゼーションタイプはサポートされていません。

### 標準総母集団レポート指標

デフォルトでは、システムには3つの標準合計母集団レポート指標が含まれており、接続にプロファイルデータセットが含まれているデータビューで使用できます。

* **合計人数（プロファイル）**: プロファイルデータセット内の合計人数。
* **合計アカウント （プロファイル）**: プロファイルデータセット内のアカウントの合計数。 [!BADGE B2B Edition]{type=Informative url="https://experienceleague.adobe.com/ja/docs/analytics-platform/using/cja-overview/cja-b2b/cja-b2b-edition" newtab=true tooltip="Customer Journey Analytics B2B Edition"}
* **合計人数（プロファイル + イベント）**：プロファイル範囲の人物とイベント範囲の人物を組み合わせた和集合カウント。

プロファイルデータセットのフィールドを使用して、3つのスコープのいずれかを持つカスタム指標を作成することもできます。

## 要件、前提条件、考慮事項

母集団全体のレポートが適切に機能するためには、前提条件、要件および考慮事項を考慮する必要があります。

### 接続要件

合計母集団レポートをサポートする接続の場合：

* 接続には少なくとも1つのイベントデータセットが必要です。 プロファイルのみの接続はサポートされていません。
* 少なくとも1つのプロファイルデータセットを接続に追加する必要があります。 合計母集団レポート指標は、イベントデータのみを含む接続に基づいて構築されたデータビューには表示されません。
* 共有ルックアップは、プロファイルデータセットごとに設定する必要があります。 共有ルックアップは、一致するキー、名前空間（ID マップフィールドの場合）、結合パスを指定することで、各プロファイルデータセットが接続内のイベントに結合される方法を定義します。

#### プロファイルデータセット設定

プロファイルデータセットが接続に追加されると、Customer Journey Analyticsは、データセットタイプに基づくデフォルトの共有ルックアップ設定を入力します。

* 人物プロファイルデータセットの場合：デフォルトは[!UICONTROL 人物]に設定されたコンテナごとの一致で、ID マップがキーフィールドとして設定されます。 このデフォルトを編集できます。 例えば、プライマリキーではなくID マップから特定の名前空間を選択します。 または、最初の名前空間が入力されていない場合（ステッチされたデータセットで一般的な場合）のセカンダリ名前空間を指定します。
* アカウントプロファイルデータセット [!BADGE B2B edition]{type=Informative url="https://experienceleague.adobe.com/ja/docs/analytics-platform/using/cja-overview/cja-b2b/cja-b2b-edition" newtab=true tooltip="Customer Journey Analytics B2B Edition"}の場合、デフォルトは[!UICONTROL Account] （グローバルアカウントが接続上で有効になっている場合は[!UICONTROL &#x200B; グローバルアカウント &#x200B;]）に設定されたコンテナごとに一致します。 アカウントフィールドには、単一のIDまたはID マップを指定できます。 アカウントフィールドがID マップの場合は、使用する名前空間を選択します。

1つのプロファイルデータセットで複数の共有ルックアップを設定して、イベントへの複数の結合パスをサポートできます。 複数の共有ルックアップで同じID マップをキーフィールドとして使用する場合、名前空間の選択は一貫している必要があります。

### データビューの要件

合計母集団レポート指標がデータビューで正しく機能するには：

* 接続には、プロファイルデータセットを含める必要があります（[接続要件](#connection-requirements)を参照）。 接続から最後の残りのプロファイルデータセットを削除すると、そのデータセットから構築されたデータビューで合計母集団レポート指標を使用できません。
* [&#x200B; データビューレベルのセグメント &#x200B;](/help/data-views/create-dataview.md#settings-segments)は、イベントを明示的にすることはできません。 データビューに直接適用されたセグメントが、イベント範囲の条件（例：`hit where page = X`）で完全に定義されている場合、そのデータビューで合計母集団レポートを作成することはできません。 総母集団レポート指標に依存する前に、任意のデータビューレベルのセグメントがプロファイルスコープのレポートと互換性があることを検証します。
* 指標の範囲を正しく設定する必要があります。 データビュービルダーでカスタム指標コンポーネントを作成する場合、フィールドのデータセットと指標の動作に基づいて、適切なスコープ（イベント、プロファイル、プロファイル + イベント）を選択します。 指標をオーディエンスまたは定期的なレポートで使用した後は、それらの依存関係を壊さずに範囲を変更することはできません。

### Workspaceとの互換性

合計母集団レポート指標は、ほとんどのWorkspace コンテキストでイベントベースの指標と一緒に使用できます。例：[&#x200B; フリーフォームテーブル &#x200B;](/help/analysis-workspace/visualizations/freeform-analysis-visualizations.md)、[行](/help/analysis-workspace/visualizations/line.md)、[&#x200B; バー](/help/analysis-workspace/visualizations/bar.md)および[水平バー](/help/analysis-workspace/visualizations/horizontal-bar.md) ビジュアライゼーション、[&#x200B; コホートテーブル &#x200B;](/help/analysis-workspace/visualizations/cohort-table/cohort-analysis.md)適切に設定された場合など。 一部のビジュアライゼーションタイプは、イベントシーケンスに本質的に依存するため、サポートされていません。

* [フォールアウト](/help/analysis-workspace/visualizations/fallout/fallout-flow.md)
* [フロー](/help/analysis-workspace/visualizations/c-flow/flow.md)
* 公開前に確認する、サポートされていない追加のビジュアライゼーションタイプ。

サポートされていないビジュアライゼーションに合計母集団レポート指標を追加すると、Workspaceはそのコンテキストで指標を使用できないことを示します。

### オーディエンスの考慮事項

総母集団レポート指標に基づいて構築されたオーディエンスは、データビューに残っている指標によって異なります。

* 総母集団レポート指標を使用する定期的なオーディエンスは失敗し、総母集団レポート指標がデータビューから削除された場合、ERROR状態に移行します。
* Customer Journey Analytics インターフェイスは、アクティブな繰り返しオーディエンスが指標に依存している間、指標の削除を防ぎ、削除を確認する前に依存関係に関するガイダンスを表示します。

### 権限

確認するには：合計母集団レポート指標が表示される前に、お客様のIMS組織または製品プロファイルに対する役割または機能のアクセス要件が表示されます。 公開前にPMにフラグを立てる価値があります。
